Frequently Asked Questions

What platforms are supported?

Microsoft Store (WACK-compliant), Windows desktop .ico, macOS, Android Play Store, iOS App Store, Chrome Extension, Favicon, Social/OG images, and Flatpak. Each platform includes all required icon sizes and store listing graphics.

Is my image uploaded to a server?

No. All image processing happens locally in your browser using the Canvas API. Your image never leaves your device — nothing is uploaded anywhere.

What image size should I use?

At least 1024×1024 pixels for best results, since some assets (like iOS App Store icons) require that resolution. Larger images will be scaled down without quality loss. PNG is preferred for transparency support.

What's the difference between Letterbox and Crop?

Letterbox (contain) fits your entire image inside the target dimensions, adding transparent padding if the aspect ratio doesn't match. Crop (cover) fills the entire target area, cutting off edges if needed. For square icons, both produce the same result.

How do Windows Store badge logos work?

Badge logos must pass Microsoft's WACK validation: all visible pixels must be pure white on a transparent background. This tool automatically detects and removes the background using dominant color detection and Otsu thresholding, then converts the foreground to white.

What is the "Flutter naming convention" option?

When enabled, iOS and macOS icons are exported with the filenames that Flutter expects (e.g. Icon-App-60x60@2x.png instead of iPhone-120.png). This lets you drop the generated files directly into your Flutter project's asset directories without renaming.

What about macOS and Windows .ico?

The macOS platform generates all 7 standard app icon sizes (16px to 1024px). The Windows platform includes a Desktop App Icon (.ico) section that generates a multi-resolution .ico file containing 16, 32, 48, 64, 128, and 256px sizes — ready for use as a Windows desktop app icon (e.g. Flutter's Windows runner).
